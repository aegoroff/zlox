# zlox

**zlox** is a Zig implementation of the Lox programming language from [Crafting Interpreters](https://craftinginterpreters.com/). It implements a bytecode compiler and virtual machine — the clox chapter of the book.

## Features

- Full Lox language: functions, closures, classes, inheritance, `super`, and `this`
- Bytecode compiler and stack-based VM
- Mark-and-sweep garbage collector
- Native functions: `clock`, `min`, `max`, `sqrt`
- Colored compile-time diagnostics ([fehler](https://github.com/ciathefed/fehler))
- [mimalloc](https://github.com/microsoft/mimalloc) as the default heap allocator

## Requirements

- [Zig](https://ziglang.org/) 0.16.0 or later

## Building

```bash
zig build
```

The binary is installed to `zig-out/bin/zlox`.

Build options:

| Flag | Default | Description |
|------|---------|-------------|
| `-Doptimize=ReleaseFast` | `Debug` | Optimization level |
| `-Dmimalloc=false` | `true` | Use the C allocator instead of mimalloc |
| `-Dversion=0.1.0` | `0.1.0-dev` | Version string shown in `--help` |

Examples:

```bash
zig build -Doptimize=ReleaseFast
zig build -Dmimalloc=false
```

## Usage

```bash
# Run a Lox file
zlox program.lox

# Read source from stdin
echo 'print "hello";' | zlox

# Disassemble generated bytecode
zlox --printcode program.lox
```

Exit codes follow the Crafting Interpreters convention: `65` for compile errors and `70` for runtime errors (in release builds).

## Testing

Run the Zig unit tests:

```bash
zig build test --summary all
```

Run the [Crafting Interpreters reference test suite](https://github.com/munificent/craftinginterpreters/tree/master/test):

```bash
./test-lox.sh              # all runnable tests
./test-lox.sh variable     # filter by path prefix
```

Environment variables for `test-lox.sh`:

| Variable | Default | Description |
|----------|---------|-------------|
| `CRAFTING_INTERPRETERS` | `~/code/craftinginterpreters` | Path to the reference repo |
| `ZLOX` | `./zig-out/bin/zlox` | Path to the zlox binary |

## Benchmarks

Compare zlox against the reference clox implementation using [poop](https://github.com/andrewrk/poop):

```bash
./bench.sh test/benchmark/fib.lox
./bench_all.sh   # all benchmark tests except zoo_batch and string_equality
```

Adjust the clox path inside `bench.sh` if needed.

## Project layout

```
src/
  main.zig           Entry point and file/stdin handling
  configuration.zig  CLI parsing (yazap)
  scanner.zig        Lexer
  compiler.zig       Parser and bytecode compiler
  chunk.zig          Bytecode format and disassembler
  vm.zig             Virtual machine
  value.zig          Lox value types and object model
  memory.zig         Heap and mark-and-sweep GC
  table.zig          Hash table for globals and methods
  builtin.zig        Native functions
  error.zig          Error types and exit codes
  mimalloc_allocator.zig  mimalloc integration
  vm_test.zig        VM integration tests
```

## Dependencies

Managed via `build.zig.zon`:

- [yazap](https://github.com/prajwalch/yazap) — CLI argument parsing
- [fehler](https://github.com/ciathefed/fehler) — diagnostic reporting
- [mimalloc](https://github.com/microsoft/mimalloc) — heap allocator (optional)

## Known limitations

Two limit tests from the reference suite are not enforced yet:

- `test/limit/no_reuse_constants.lox`
- `test/limit/too_many_constants.lox`

All other runnable reference tests pass.
